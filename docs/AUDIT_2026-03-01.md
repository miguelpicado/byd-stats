## 1. Seguridad (52/100) [anterior: 38/100]

### 1.1 Hallazgos CRITICOS

#### S-04: Google Client ID Expuesto (SIN CORREGIR)
- **Severidad:** CRITICA
- **Archivo:** `android/app/src/main/res/values/strings.xml` linea 7
- **Problema:** OAuth Client ID hardcodeado: `REDACTED_GOOGLE_CLIENT_ID`
- **Riesgo:** Ataques de sustitucion de tokens, flujos OAuth no autorizados
- **Fix:** Mover a `BuildConfig` e inyectar en build time

#### S-NEW-01: Clave de Cifrado Hardcodeada en Codigo Fuente
- **Severidad:** CRITICA
- **Archivo:** `src/services/firebase.ts` linea 6
- **Problema:** Clave AES-256 en texto plano en el codigo:
  ```typescript
  const ENCRYPTION_KEY = 'd0a5edbd5edc9a1bb954e16cdb4c9391673081a5e0e44554018b4fbd08889661';
  ```
- **Riesgo:** Si el repositorio se compromete, todos los datos cifrados pueden ser descifrados
- **Fix:** Mover a Firebase Cloud Functions config o Firebase Secret Manager

### 1.2 Hallazgos ALTOS

#### S-05: Datos Sensibles en Logs (PARCIALMENTE CORREGIDO)
- **Severidad:** ALTA
- **Archivos:**
  - `WearableMessageListenerService.kt` linea 104: `Log.i("WearableService", "Triggering native $action ($functionName) for VIN: $vin (User: ${user.uid})")`
- **Progreso:** WearSyncPlugin.java ya no loguea el VIN en claro (ahora dice "encrypted")
- **Pendiente:** WearableMessageListenerService.kt sigue logueando VIN y UID sin redactar
- **Fix:** Redactar VIN (`***${vin.takeLast(4)}`) y UID

#### S-NEW-02: Mixed Content Permitido en Capacitor
- **Severidad:** ALTA
- **Archivo:** `capacitor.config.json` linea 10
- **Problema:** `"allowMixedContent": true`
- **Riesgo:** Permite cargar contenido HTTP en contexto HTTPS, habilitando ataques MITM
- **Fix:** Establecer `false` en produccion, usar configuracion separada para desarrollo

#### S-NEW-03: Sin Rate Limiting en Cloud Functions
- **Severidad:** ALTA
- **Archivo:** `functions/src/bydFunctions.ts` (todas las funciones callable)
- **Problema:** Ninguna funcion tiene limite de peticiones por usuario
- **Riesgo:** Ataques de fuerza bruta, DoS, abuso de la API BYD
- **Fix:** Implementar rate limiting con contadores Firestore por UID con TTL

### 1.3 Hallazgos MEDIOS

#### S-NEW-04: Validacion Insuficiente en Webhook MQTT
- **Archivo:** `functions/src/bydFunctions.ts` lineas 1099-1118
- **Problema:** El webhook recibe datos con validacion minima (solo verifica que `event` exista)
- **Fix:** Validar esquema completo con zod, validar formato VIN (17 chars), limitar tamano de payload

#### S-NEW-05: Webhook Sin HMAC Signing
- **Archivo:** `functions/src/bydFunctions.ts` lineas 1079-1126
- **Problema:** Solo usa header `x-webhook-secret` para autenticacion
- **Fix:** Implementar HMAC-SHA256 signing como GitHub webhooks, validar timestamp contra replay attacks

#### S-NEW-06: Sin Limite de Tamano en FileOpenerPlugin
- **Archivo:** `android/app/src/.../plugins/FileOpenerPlugin.java` lineas 43-52
- **Problema:** Carga archivos completos en memoria sin limite, luego los codifica en Base64 (duplica tamano)
- **Fix:** Validar tamano maximo (50MB), usar streaming para archivos grandes

#### S-NEW-07: Console Logs en Cloud Functions
- **Archivo:** `functions/src/googleMaps.ts` lineas 28-47
- **Problema:** Logging verbose con patrones de procesamiento de datos
- **Fix:** Usar logging estructurado con niveles apropiados

### 1.4 Lo que esta BIEN
- `WebView.setWebContentsDebuggingEnabled(BuildConfig.DEBUG)` correctamente condicionado
- Keystore passwords en env vars / local.properties (no en codigo)
- `minifyEnabled true` + `shrinkResources true` + ProGuard habilitado
- `EncryptedSharedPreferences` con AES-256-GCM para datos sensibles
- `android:allowBackup="false"` correctamente configurado
- `usesCleartextTraffic="false"` en manifest
- Network security config presente
- Source maps deshabilitados en produccion
- FileProvider paths restringidos a directorios especificos
- Console logs al logcat condicionados a DEBUG
- `Handler.postDelayed()` en lugar de `Thread.sleep()`
- No hay `dangerouslySetInnerHTML` ni XSS detectado
- No hay `eval()` ni `Function()` constructor
- No hay inyeccion SQL (sql.js usa queries parametrizadas)
- No hay `addJavascriptInterface` (evita inyeccion JS nativa)
- Intent filters restringidos a MIME types especificos
- Credenciales Firebase cifradas con AES-256-GCM
- Firestore rules bloquean `private/credentials`
- Auth + ownership checks en Cloud Functions
